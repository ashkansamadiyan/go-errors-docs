Welcome to the **go-errors** documentation! This library brings Go-style error handling to JavaScript/TypeScript, allowing you to handle errors elegantly without try-catch blocks, using a functional approach that's both type-safe and intuitive.

## Key Features

- 🎯 **Type-safe** error handling with full TypeScript support
- 🔄 **Go-style** result tuples `[value, error]`
- ⚡ **Zero dependencies** and lightweight
- 🔀 **Unified API** for both synchronous and asynchronous operations
- 🌐 **Enhanced Fetch** with built-in transformers and type safety
- 🛡️ **Predictable error handling** without try-catch blocks
- 📦 **Tree-shakeable** and optimized for bundle size

## Quick Installation

```bash
bun add go-errors
```

## Basic Usage

```typescript
import { goSync, goFetch } from 'go-errors';

// Synchronous usage
let [value, err] = goSync(() => {
  // Your code that might throw
  return "success";
});

if (err) {
  console.error("Something went wrong:", err.message);
} else {
  console.log("Got value:", value);
}

// Asynchronous usage with goFetch
let [data, err] = await goFetch<User>('/api/user/123', {
  responseTransformer: (data) => ({
    ...data,
    fullName: `${data.firstName} ${data.lastName}`
  })
});

if (err) {
  console.error("API call failed:", err.message);
} else {
  console.log("User data:", data);
}
```

## Why go-errors?

Traditional error handling in JavaScript/TypeScript often relies on try-catch blocks, which can lead to:
- Verbose and nested code structures
- Easy-to-miss error cases
- Mixing of error handling and business logic
- Type safety challenges

**go-errors** solves these problems by:
1. Providing a consistent, functional approach to error handling
2. Ensuring type safety through TypeScript
3. Separating error handling from business logic
4. Making error cases explicit and impossible to ignore
5. Supporting both synchronous and asynchronous operations with a unified API
6. Providing powerful fetch capabilities with built-in transformers
